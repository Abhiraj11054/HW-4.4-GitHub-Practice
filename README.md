# HW-4.4-GitHub-Practice

## About This Project
This repository is for HW 4.3, where I am learning and demonstrating how to use GitHub for version control. The project focuses on using branches, issues, commits, and pull requests to manage a full workflow.

## Project Plan
The plan for this repository is to:
- Use Git branches to separate work into different development areas
- Track progress using GitHub Issues
- Make regular commits to document changes
- Use pull requests to merge work into the main branch
- Store all required HW 4.3 files in the repository (QMD and PDF)

## Repository Organization
- `main` branch: final stable version of the project
- `hw4-dev` branch: development work and file creation
- `hw4-issues` branch: planning and issue-related work
- Issues tab: used to track tasks and progress
- Pull Requests: used to merge completed work into main

## Files Included
- README.md (this file)
- HW 4.3 QMD file
- HW 4.3 PDF submission
- GitHub Issues (2 total)
- Commit history showing project progress
